SELECT * FROM users ORDER BY id LIMIT 1 OFFSET 14; The "15th User" Query. `OFFSET 14` skips the first 14 rows. `LIMIT 1` takes the very next row (the 15th). Crucial for pagination. SELECT * FROM t1 UNION SELECT * FROM t2; Combines the result sets of two queries. UNION performs a sort and removes duplicates (Slower). SELECT * FROM t1 UNION ALL SELECT * FROM t2; Combines result sets but keeps duplicates. It is much faster because it just appends the data without sorting. INNER JOIN SELECT * FROM users u JOIN orders o ON u.id=o.uid;
Returns rows ONLY when there is a match in BOTH tables. Users with no orders are hidden. LEFT JOIN (Troubleshooting) SELECT * FROM users u LEFT JOIN orders o ON u.id=o.uid;
Returns ALL rows from the Left table (Users), even if there are no matches in the Right (Orders). RIGHT JOIN Returns ALL rows from the Right table, and matched rows from the Left. Rarely used in daily Ops. FULL OUTER JOIN SELECT * FROM t1 FULL OUTER JOIN t2 ON t1.id=t2.id
Returns EVERYTHING. Rows that match, plus rows from Left with no match, plus rows from Right with no match. EXISTS SELECT * FROM u WHERE EXISTS (SELECT 1...);
Optimized filter. Checks if "at least one" related row exists. Faster than IN for large lists. Window Function ROW_NUMBER() OVER (PARTITION BY dept ORDER BY salary)
Assigns a rank (1, 2, 3...) to rows within a specific group (Partition). 		DROP TABLE old_logs; DDL (Data Definition Language). Permanently deletes the table and its structure. Cannot be undone easily. DELETE FROM users WHERE id=15; DML (Data Manipulation Language). Removes a specific row of data. The table structure remains intact. ALTER TABLE users ADD COLUMN phone TEXT; DDL. Modifies the structure of an existing table to add a new field/column. TRUNCATE TABLE temp_data; DDL. Instantly empties all data from a table but keeps the structure. Much faster than DELETE for clearing tables. EXPLAIN ANALYZE SELECT ... ; Runs the query and shows the actual execution time and cost. Essential for debugging slow queries. SELECT pid, state, query FROM pg_stat_activity; Postgres Specific. Shows all currently running queries. Use this to find queries that are "stuck" or running for too long. VACUUM (VERBOSE, ANALYZE); Postgres Specific. Reclaims storage space from dead rows and updates the query planner statistics for better performance. UPSERT INSERT INTO ... ON CONFLICT (id) DO UPDATE ...
Tries to Insert a row. If the Primary Key already exists, it Updates the existing row instead. Transaction Hygiene BEGIN; UPDATE...; COMMIT;
Ensures Atomicity. Either all commands in the block succeed (Commit), or none of them do (Rollback).

Class vs Object Class: The Blueprint/Template. Object: The specific instance built from that blueprint. int vs Integer int: Primitive type, stores raw values, cannot be null. Integer: Wrapper Class, is an Object, can be null. Inheritance Mechanism where a new Child class reuses (inherits) fields and logic from a Parent class. Polymorphism "One name, many forms". Overloading: Same method name, different parameters. Overriding: Child class replaces Parent method. IOC (Inversion of Control) The Spring Framework manages the lifecycle of objects (creation, destruction) so the developer doesn't have to (`new Object()`). DI (Dependency Injection) Spring injects the required dependencies into a class automatically using annotations like `@Autowired`. @RestController Spring Annotation. Marks a class as a controller where every method returns a domain object instead of a view (JSON API). @Service Marks a class as holding Business Logic. This is where calculations and data processing happen. @Repository Marks a class as a Data Access Object (DAO). It interacts directly with the Database (Hibernate/JPA). Maven (pom.xml) Project Management tool. `pom.xml` defines dependencies (libraries/jars) and how to build/package the app. kubectl get pods -n <ns> Lists all Pods in a Namespace. Shows Status (Running/Error), Age, and Restart Counts. kubectl describe pod <name> The #1 Debug Tool. Shows configuration and the "Events" stream. Use this to find why a pod is Pending or Crashing. kubectl logs -f <pod> Streams the `stdout` (standard output) from the container. This is where you see Application Stack Traces. kubectl top nodes Shows current CPU and Memory usage for all nodes in the cluster. identifying resource exhaustion. Deployment Manages stateless apps. Ensures the desired number of replicas are always running. StatefulSet Manages stateful apps (like Databases). Pods get persistent IDs (db-0, db-1) and sticky storage. Service / Ingress Service: Internal Load Balancer (ClusterIP). Ingress: External Route (HTTP/HTTPS) handling traffic into the cluster. ConfigMap / Secret Injects configuration (URLs, properties) and Sensitive Data (Passwords, Keys) into Pods as Env Variables. PV / PVC Persistent Volume: Actual storage. Claim: A request for storage by a Pod. Ensures data survives Pod restarts. Control Plane The "Brain" of K8s. Includes API Server (Frontend), ETCD (Database), Scheduler (Placement), and Controller Manager. EC2 / S3 / IAM / VPC The Core 4. Compute (Servers), Storage (Objects), Identity (Permissions), and Networking (Isolation). IAM Role An identity that can be assumed by a machine (EC2/Lambda). It provides temporary credentials, removing the need for hardcoded keys. Security Group (SG) Stateful Firewall at the Instance level. If you allow Traffic IN, the response is automatically allowed OUT. NACL (Network ACL) Stateless Firewall at the Subnet level. You must explicitly allow traffic both IN and OUT. ALB / Target Group ALB: Layer 7 Load Balancer (HTTP). Target Group: Performs Health Checks on instances to know where to send traffic. VPC Endpoint Allows private instances to connect to AWS Services (like S3) without needing an Internet Gateway or NAT. CloudWatch AWS's native monitoring service. Collects Logs, Metrics (CPU/Disk), and triggers Alarms. Route 53 AWS Managed DNS. Translates domain names to IP addresses. Supports Health Checks and Failover routing. Lambda Serverless Compute. You upload code, AWS runs it. You pay only for compute time used (ms). Auto Scaling Group Automatically adjusts the number of EC2 instances up or down based on load metrics (e.g., Average CPU). CloudWatch Metrics Key metrics to watch: CPUUtilization, DiskReadOps, ELB Latency, 5xxErrorCount, StatusCheckFailed. Splunk / Nieman Log Aggregation tools. Use SPL queries (`index=prod "Error"`) to find patterns across thousands of servers. Postman / Curl Tools to interact with APIs directly. 1. DispatcherServlet The "Front Controller". It receives ALL incoming HTTP requests and delegates them to the right handler. 2. Handler Mapping Scans the code to find which @Controller method matches the URL (e.g., `/api/users`). 3. Service Layer Controller calls the @Service. This contains the Business Logic (calculations, validations). 4. Repository Layer Service calls the @Repository. This layer uses Hibernate/JPA to query the Database. 5. Database The SQL query executes. The DB returns raw data, which Hibernate converts into Java Objects (Model). 6. Response The Controller takes the data, serializes it to JSON, and sends a 200 OK response to the client. Exception Handling If any step fails, @ControllerAdvice catches the crash and returns a friendly error (400/500 JSON). #!/bin/bash Standard Shebang line. `set -Eeuo pipefail` ensures the script exits immediately if any command fails. Compresses old logs into a `.tar.gz` archive to save disk space. truncate -s 0 "$LOG/app.log" Critical Trick: Clears the content of the active log file without deleting the file itself. Trace API Use Postman or Curl to reproduce the 500/404 error locally. Check the response body for clues. 3. Check Service Health Run `systemctl status tomcat`. Is it Active, Inactive, or Failed? 4. Check Resources Run `top` (CPU/RAM) and `df -h` (Disk). A 100% full disk is a common cause of crashes. 5. DB & Java Deep-Dive Check Database connectivity. Run `jstack` to see if Java threads are locked or waiting. 6. K8S Check Run `kubectl describe pod` to check for Events (BackOff, OOMKilled). Check `kubectl logs`. 7. Network Check Security Groups (is port 8080 allowed?), Load Balancer Target Health, and DNS resolution. 8. RCA Once fixed, perform Root Cause Analysis. Document the "Why" and the "Fix" in a Jira ticket. SLA (Service Level Agreement) The contract with the customer (e.g., "We guarantee 99.9% Uptime"). The deadline you must meet. MTTR (Mean Time To Recovery) The average time it takes to fix a broken system. Lower is better. The #1 metric for SREs. RCA (Root Cause Analysis) The post-incident report. Details exactly what broke, why it broke, and how to prevent it. SME (Subject Matter Expert) The person who knows the specific application logic best. Usually a developer or senior ops. Bridge Call A live conference call (Zoom/Teams) opened during P1 incidents to coordinate multiple teams (DB, Net, App). Handover The process of transferring knowledge, active tickets, and incident status from one shift (India) to the next (US). SOP (Standard Operating Procedure) A step-by-step document describing exactly how to perform a routine task or fix a known issue.

200 OK / 201 Created Success. No action needed. The request was received and processed correctly. 400 Bad Request Client Error. The user sent invalid JSON or missing fields. Action: Check the payload sent. 401 Unauth / 403 Forbidden Auth Error. Token expired (401) or valid token but no permission (403). Action: Check IAM/RBAC. 404 Not Found Routing Error. Wrong URL or the Application (WAR) is not deployed. Action: Check paths and deployment status. 500 Internal Error Server Error. The Java code crashed. Action: Immediately check `catalina.out` for the Exception Stack Trace. 503 Service Unavailable Capacity Error. Server is overloaded or down. Action: Check CPU/Memory and Scaling limits. 504 Gateway Timeout Timeout. The App is waiting too long for the Database or a 3rd Party API. Action: Check downstream health. bin/startup.sh Manual script to start Tomcat if the systemctl service fails or is not configured. catalina.out The MASTER LOG file for Tomcat. 99% of Java application errors will be found here. work/ & temp/ Temporary cache directories. If an app acts weird after deployment, delete these and restart to clear the cache. setenv.sh Configuration file to set environment variables, specifically the Java Heap Size (`-Xmx4g -Xms4g`). server.xml Main Tomcat configuration file. Defines which Ports (8080) and Connectors the server listens on. systemctl restart tomcat The standard, safe, and automated way to restart the application in a production environment. ps -ef | grep tomcat Verifies if the Tomcat process ID (PID) is actually running.
