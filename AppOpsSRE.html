SELECT * FROM t1 UNION SELECT * FROM t2;Combines rows; REMOVES duplicates.
SELECT * FROM t1 UNION ALL SELECT * FROM t2;Combines rows; KEEPS all duplicates (Faster).
INNER JOIN: SELECT * FROM u JOIN o ON u.id=o.uid;Matches records present in BOTH tables.
LEFT JOIN: SELECT * FROM u LEFT JOIN o ON u.id=o.uid;Finds ORPHANS (Users without Orders).
RIGHT JOIN: SELECT * FROM u RIGHT JOIN o ON u.id=o.uid;Keeps all Right table rows (Rarely used).
FULL OUTER: SELECT * FROM t1 FULL OUTER JOIN t2 ...Keep EVERYTHING (Match + No Match). Reconciliation.
EXISTS: SELECT * FROM u WHERE EXISTS (SELECT 1...);Fast filter "at least one related row".
Window: ROW_NUMBER() OVER (PARTITION BY dept ORDER BY salary)Ranks items within a category.
Agg: SELECT count(*), status FROM orders GROUP BY status;Counts items by category.
DROP TABLE old_logs;DDL: Deletes table structure permanently.
DELETE FROM users WHERE id=15;DML: Removes specific data row (User 15).
ALTER TABLE users ADD COLUMN phone TEXT;DDL: Modifies schema to add a new field.
TRUNCATE TABLE temp_data;DDL: Empties table data fast but keeps structure.
EXPLAIN ANALYZE SELECT ... ;Debugs slow queries by showing execution cost.
SELECT pid, state, query FROM pg_stat_activity;Finds stuck/long-running queries in DB.
VACUUM (VERBOSE, ANALYZE);Reclaims storage space and updates DB stats.
UPSERT: INSERT ... ON CONFLICT (id) DO UPDATE ...Insert or Update if exists.
LOCKS: SELECT pid, mode FROM pg_locks...Identifies queries blocking others.
TXN: BEGIN; UPDATE...; COMMIT;Ensures atomic operations (all or nothing).

Q: "500 vs 504?"A: 500=App Crash (Code); 504=Gateway Timeout (DB slow).
Q: "Fix Unhealthy LB?"A: 1.Check SG
2.Check App Port (`ss`)
3.Check `/health` path.
Q: "UNION vs ALL?"A: UNION removes dupes (slower); ALL keeps dupes (faster).
Q: "Left vs Inner?"A: LEFT finds orphans (missing data); INNER finds matches.
Q: "Automation Example?"A: "I scripted log rotation using `tar` & `truncate` to stop disk alerts."
Q: "Handle P1?"A: 1.Ack 2.Bridge 3.Mitigate(Restart) 4.Communicate 5.RCA.
