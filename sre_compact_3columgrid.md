| `w` – shows who is logged in, uptime & load average.                                                                          | `SELECT * FROM users ORDER BY id LIMIT 1 OFFSET 14;` – “15th User”.                                                            | `kubectl get pods -n <namespace>` – list pods with status & restart count.                                                    |
| `uname -a` – kernel version, architecture & OS info.                                                                          | `SELECT * FROM t1 UNION SELECT * FROM t2;` – combine rows **removing duplicates**.                                            | `kubectl describe pod <pod>` – full pod spec, events, container states.                                                    |
| `hostnamectl` – view / change hostname, OS release, virtualization.                                                          | `SELECT * FROM t1 UNION ALL SELECT * FROM t2;` – combine rows **keeping duplicates**.                                        | `kubectl logs -f <pod>` – stream live container logs (stack traces).                                                       |
| `systemctl status <svc> && systemctl restart <svc>` – check service health; restart if hung.                                 | `INNER JOIN` – `SELECT u.*, o.* FROM users u JOIN orders o ON u.id = o.user_id;` – rows present in **both** tables.          | `kubectl top nodes` / `kubectl top pods` – CPU/Memory usage per node or pod (detect hot spots).                           |
| `journalctl -u <svc> -xe` – deep dive into service logs.                                                                      | `LEFT JOIN` – `SELECT u.*, o.* FROM users u LEFT JOIN orders o ON u.id = o.user_id;` – find users **without** orders.        | `kubectl exec -it <pod> -- /bin/bash` – open a shell inside a pod for ad‑hoc debugging.                                   |
| `chmod 755 <file>` – set rwxr‑x‑r‑x permissions.                                                                              | `RIGHT JOIN` – `SELECT u.*, o.* FROM users u RIGHT JOIN orders o ON u.id = o.user_id;` – keep all rows from the **right** table. | `kubectl rollout status deployment/<deployment>` – wait for a rollout to finish (zero unavailable).                     |
| `chown -R user:group <dir>` – recursively fix permission errors.                                                             | `FULL OUTER JOIN` – `SELECT * FROM t1 FULL OUTER JOIN t2 ON t1.id = t2.id;` – return **all** rows from both tables.          | `kubectl apply -f <manifest.yaml>` – create / update Deployments, Services, ConfigMaps, Secrets, etc.                    |
| `find /path -mtime +7 -delete` – delete files older than 7 days (log‑rotation).                                              | `EXISTS / SUB‑QUERY` – `SELECT * FROM users u WHERE EXISTS (SELECT 1 FROM orders o WHERE o.user_id = u.id);` – fast filter. | `kubectl get svc -n <ns>` – list Services (ClusterIP, NodePort, LoadBalancer).                                            |
| `grep -R "ERROR" /var/log` – recursive search for error strings in logs.                                                     | **UPSERT** – `INSERT INTO users(id,name) VALUES (15,'Bob') ON CONFLICT (id) DO UPDATE SET name = EXCLUDED.name;` – insert‑or‑update. | `aws ec2 describe-instances --filters "Name=tag:Env,Values=prod"` – list EC2 instances in production.                     |
| `ss -tuln` – fast view of all listening TCP/UDP ports.                                                                        | `VACUUM (VERBOSE, ANALYZE);` – reclaim storage & update PostgreSQL statistics.                                             | `aws s3 ls s3://my-bucket/ --recursive` – list objects in an S3 bucket (useful for checking lifecycle rules).           |
| `netstat -tulpn` – ports + owning PID.                                                                                       | `EXPLAIN ANALYZE SELECT …;` – show execution plan & cost for a query (debug slow queries).                                 | `aws cloudwatch get-metric-statistics --namespace AWS/EC2 --metric-name CPUUtilization --statistics Average …` – fetch CPU % over a period. |
| `lsof -i :8080` – which process holds port 8080?                                                                              | `SELECT pid, state, query FROM pg_stat_activity;` – list running PostgreSQL queries (spot long‑running).                  | `aws iam get-role --role-name <role>` – view IAM role trust policy & permissions (useful for SG/IRSA checks).          |
| `telnet <IP> <Port>` – test TCP connectivity.                                                                                | `SELECT pid, mode FROM pg_locks l JOIN pg_stat_activity a ON l.pid = a.pid;` – identify blocking locks.                    | `aws rds describe-db-instances --db-instance-identifier <id>` – check RDS health, endpoint, storage.                     |
| `nc -zv <IP> <Port>` – netcat stealth port scan (faster than telnet).                                                       | `TRUNCATE TABLE temp_data;` – fast empty of a table while keeping its structure.                                           | `aws elb describe-load-balancers --load-balancer-names <alb>` – view ALB health‑check configuration.                    |
| `curl -I -v <URL>` – fetch HTTP headers & verbose output.                                                                    | `SELECT pid, state, query FROM pg_stat_activity;` – spot stuck/long‑running queries (repeat for emphasis).                | `aws cloudwatch put-metric-alarm --alarm-name HighCPU --metric-name CPUUtilization --threshold 80 --comparison-operator GreaterThanThreshold …` – create a CPU alarm. |
| `traceroute <IP>` – show each hop to the destination.                                                                        | `INSERT … ON CONFLICT … DO UPDATE` – upsert pattern (idempotent writes).                                                   | `aws route53 list-hosted-zones` – list DNS zones; `aws route53 test-dns-answer …` – verify DNS resolution.               |
| `nslookup <domain>` / `dig <domain> ANY` – DNS lookup & detailed records.                                                    | `SELECT * FROM users u LEFT JOIN orders o ON u.id = o.user_id WHERE u.id = 15;` – find missing orders for user 15.          | `aws logs put-metric-filter --log-group-name /aws/ecs/container-logs/app-prod --filter-name ExceptionCount …` – count exceptions in logs. |
| `ip route show` – display routing table (gateway, interfaces).                                                               | `SELECT * FROM users u WHERE EXISTS (SELECT 1 FROM orders o WHERE o.user_id = u.id);` – “at least one related row”.        | `aws autoscaling describe-auto-scaling-groups --auto-scaling-group-names <asg>` – view ASG scaling policies.            |
| `grep -R "ERROR" /var/log` – recursive error search (same as column 1, repeated for emphasis).                               | `SELECT * FROM users u WHERE NOT EXISTS (SELECT 1 FROM orders o WHERE o.user_id = u.id);` – find users **without** orders. | `aws cloudwatch get-metric-statistics --namespace AWS/ApplicationELB --metric-name TargetResponseTime …` – monitor ALB latency. |
| `find /path -mtime +7 -delete` – delete files older than 7 days (log‑rotation).                                              | `SELECT * FROM t1 FULL OUTER JOIN t2 ON t1.id = t2.id;` – keep everything (matched + unmatched).                           | `aws cloudwatch put-dashboard --dashboard-name TomcatHealth --dashboard-body file://dashboard.json` – push a custom dashboard. |
| `chmod 755 <file>` – set rwxr‑x‑r‑x permissions.                                                                             | `ANALYZE;` – refresh PostgreSQL planner statistics (helps the optimizer).                                                   | `aws s3api put-bucket-lifecycle-configuration --bucket <bucket> --lifecycle-configuration file://lifecycle.json` – set S3 lifecycle (e.g., Glacier after 30 days). |
| `chown -R user:group <dir>` – recursively change ownership.                                                                   | `LOCKS CHECK` – `SELECT pid, mode FROM pg_locks l JOIN pg_stat_activity a ON l.pid = a.pid;` – see which queries hold locks. | `aws iam create-role --role-name <role> --assume-role-policy-document file://trust.json` – create an IRSA role for EKS pods. |
